## 1. 产品概述

### 1.1 一句话定位

**简道云是帆软软件旗下的 SaaS 零代码应用搭建平台，国内市占率第一的表单驱动型低代码产品，核心能力为"表单 → 流程 → 仪表盘"全链路。**

### 1.2 核心设计理念

- **零代码优先**: 拖拽式操作，无需编写任何代码即可搭建完整业务应用
- **表单即应用**: 以表单作为数据采集入口，流程作为业务驱动引擎，仪表盘作为数据消费终点
- **极低学习门槛**: 业务人员 3 天即可上手搭建日常管理系统
- **帆软 BI 基因**: 背靠帆软 FineReport/FineBI，数据可视化能力领先同类产品
- **全民开发**: 旗舰版定位"深度应用全民开发"，鼓励组织内非 IT 人员自主搭建

### 1.3 团队与发展历程

| 时间 | 里程碑 |
|------|--------|
| 2015 | 简道云正式上线，为国内表单类无代码应用开发工具的开创者 |
| 2017 | 正式商业化，注册用户突破 20 万，在没有 1 名正式销售的情况下实现盈亏平衡 |
| 2019 | 销售额突破 3,000 万元 |
| 2020 | 企业数字化需求集中爆发，团队开始扩张 |
| 2024 | ARR 增长近 30%，续费率超 90%，市占率 37% |
| 2024 | 引入 AI 助理、智能助手 Pro、Agent 节点等 AI 功能 |
| 2025 | IDC 报告：连续 4 年蝉联中国零代码平台整体市场第一，市占率 34.5% |

### 1.4 商业模式特点

简道云始终坚持"自负盈亏"的经营策略，这在 SaaS 行业以"以价换量"为主的竞争环境中实属异类。其核心特征：

- **不依赖 VC 烧钱**: 2017 年即实现盈亏平衡，不靠融资驱动增长
- **极致人力效率**: 2015-2020 年团队规模严格控制，保持"小团队大产品"
- **高续费率**: 金额续费率超过 100%，客户续约率超过 80%
- **健康增长**: 2024 年 ARR 增长近 30%，在 SaaS 行业普遍亏损的背景下逆势增长

### 1.5 对标关系

简道云在 AUDEBase 对标矩阵中的定位：

| 维度 | 简道云 | AUDEBase 定位 |
|------|--------|--------------|
| 架构模式 | 表单驱动（组件编排） | 模型驱动为主，表单驱动为辅 |
| 目标用户 | 业务人员/数字专员 | 开发者 + 高级业务人员 |

| 数据建模 | 表单内字段定义 + 关联字段 | Collection + Field 元数据驱动 |
| 部署 | SaaS 为主 + 私有化 | SaaS + 私有化 + 混合部署 |

## 2. 技术架构

### 2.1 整体架构

简道云采用分布式微服务架构，部署在阿里云上，通过 Docker + Kubernetes 进行容器编排和集群管理。

```
┌─────────────────────────────────────────────────┐
│                  客户端层                         │
│   PC 浏览器  │  移动端 H5  │  钉钉/企微/飞书     │
└──────────────────────┬──────────────────────────┘
                       │ SSL/TLS + WAF
┌──────────────────────┴──────────────────────────┐
│                  API 网关                         │
│         单点登录 │ 权限校验 │ 路由分发             │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────┴──────────────────────────┐
│              微服务层 (HTTP + 消息队列)            │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐         │
│  │企业通讯录 │ │数据分析  │ │数据采集  │         │
│  │ SSO/组织 │ │数据工厂  │ │表单引擎  │         │
│  │ 集成服务 │ │仪表盘    │ │流程引擎  │         │
│  └──────────┘ └──────────┘ └──────────┘         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐         │
│  │消息队列  │ │定时任务  │ │文件服务  │         │
│  │通知推送  │ │提醒/调度 │ │PDF/Excel │         │
│  └──────────┘ └──────────┘ └──────────┘         │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────┴──────────────────────────┐
│                 数据存储层                         │
│   MongoDB (企业数据) │ Redis (缓存) │ OSS (文件)  │
└─────────────────────────────────────────────────┘
```

### 2.2 技术栈明细

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 后端语言 | Node.js + Java | 混合语言微服务 |
| 容器编排 | Docker + Kubernetes | 云原生架构，支持弹性伸缩 |
| 云平台 | 阿里云 | VPC、EIP、SLB、ECS、OSS |
| 数据库 | MongoDB (结构化 + 非结构化) | 企业数据主存储 |
| 缓存 | Redis | 会话缓存、数据缓存、消息队列 |
| 文件存储 | 阿里云 OSS | 附件、图片、Excel 导出等 |
| 前端 | React/Vue (推测) | Web 端拖拽式表单设计器 |
| 通信协议 | HTTP/HTTPS、Webhook、MQTT（IoT） | 内部 HTTP + 队列，外部 Webhook/API |
| 安全传输 | SSL/TLS 全程加密 | WAF 防火墙 → 集群服务 |
| 负载均衡 | 阿里云 SLB | 入口流量分发 |

### 2.3 核心运行原理

#### 2.3.1 元数据驱动机制

简道云的低代码生成基于**元数据驱动**模式：

1. 用户通过拖拽式界面配置表单/流程/仪表盘
2. 所有配置被转化为 JSON 元数据存储
3. 运行时动态解析 JSON 元数据，生成可执行的前后端代码（Vue 组件 + Node.js 接口）

这与 NocoBase 的 Schema Engine 和 AUDEBase 规划的 D7 决策（自研轻量 Schema→UI 映射器）思路一致，区别在于简道云的元数据 schema 是**封闭的**（仅平台内部使用），而 NocoBase/AUDEBase 追求**开放的**插件可扩展 schema。

#### 2.3.2 数据联动机制

- **跨表关联**: 通过外键（主-子表关系）实现数据聚合查询，底层依赖 JOIN 优化
- **触发机制**: 数据库事件（Insert/Update/Delete）触发 Webhook 回调或智能助手内置脚本
- **公式计算**: 支持 80+ 函数，包括逻辑函数、文本函数、日期函数、数学函数等

#### 2.3.3 权限控制模型

简道云采用 **RBAC（基于角色的访问控制）**，实现三级粒度：

| 级别 | 说明 | 实现方式 |
|------|------|---------|
| 表单级 | 哪些角色可以访问某张表单 | 表单可见性设置 |
| 字段级 | 角色在某字段上的权限（可见/可编辑/隐藏） | 流程节点字段权限 + 表单字段权限 |
| 行级 | 角色只能看到符合条件的数据行 | SQL WHERE 条件注入（如 `WHERE dept_id == 用户部门`） |

这与 AUDEBase 的 D10（Record Rules）+ D11（字段级权限）设计完全一致。

## 3. 核心功能

简道云围绕企业级应用搭建场景，构建了**六大核心功能模块**，既可独立使用也可组合搭建完整业务系统。

### 3.1 表单引擎

表单是简道云的**数据入口和业务起点**。

| 能力 | 详情 |
|------|------|
| 字段类型 | 20+ 种：文本、数字、日期、下拉框、单选/多选、定位、图片、附件、子表单、关联数据、成员、部门等 |
| 操作方式 | 纯拖拽式设计，零代码配置 |
| 数据验证 | 字段级校验规则、必填/唯一性、正则表达式 |
| 计算字段 | 80+ 函数支持公式计算、跨表引用 |
| 表单样式 | 分页表单、多列布局、条件显隐 |
| 公开表单 | 生成外链，无需登录即可填写 |
| 数据联动 | 关联字段实现表单间数据引用、联动筛选 |

### 3.2 流程引擎

流程是简道云的**业务驱动核心**，将审批、通知、自动化串联起来。

| 能力 | 详情 |
|------|------|
| 流程设计 | 拖拽式节点 + 连线操作 |
| 审批节点 | 单人/多人/角色/部门审批，支持会签/或签 |
| 条件分支 | 基于表单字段值的动态分支（如金额 > 5000 → 总监审批） |
| 字段权限 | 每个流程节点独立控制字段的可见/可编辑 |
| 流程操作 | 回退、转交、加签、催办、批量审批 |
| 子流程 | 主流程触发子流程，支持跨表单业务联动 |
| 版本管理 | 流程版本控制，历史版本可追溯 |
| 流程分析 | BPA（业务流程分析）视图，统计节点耗时、流转效率 |
| 流程日志 | 完整操作留痕，支持审计追溯 |

**与 AUDEBase 的关系**: 简道云的流程引擎是审批流轻引擎（非完整 BPMN 2.0）。AUDEBase 规划的工作流引擎（Phase 4）需要覆盖审批流 + 业务流程自动化，复杂度更高。

### 3.3 仪表盘

仪表盘是简道云的**数据消费层**，承载帆软 FineReport/FineBI 的 BI 基因。

| 能力 | 详情 |
|------|------|
| 图表类型 | 20+ 种：柱形图、折线图、饼图、雷达图、地图、甘特图、组合图等 |
| 数据源 | 多表单/应用数据自动汇总分析 |
| 图表联动 | 点击一个图表的数据区域，其他图表自动过滤到对应维度 |
| 数据预警 | 基于阈值的数据自动告警（如库存低于安全库存） |
| 数据推送 | 定时推送数据报告到钉钉/企业微信/邮件 |
| 移动端适配 | 仪表盘自动适配移动端，支持数据随时查看 |
| 数据管理 | 仪表盘中可直接新增/编辑/删除数据——仪表盘不仅是"看的"，也是"用的" |

简道云的 BI 能力是相对 NocoBase 和明道云的显著优势。NocoBase 需要集成外部 BI，明道云仅提供基础报表。

### 3.4 数据工厂

数据工厂是简道云的**ETL 工具**——随着应用增多，数据分散在不同表单和应用中，数据工厂负责将分散数据整合、清洗、计算。

| 节点类型 | 能力 |
|----------|------|
| 横向连接 | 两张表左右相连（JOIN），获得新增多列的结果表 |
| 追加合并 | 多张表上下相连（UNION），获得新增多行的结果表 |
| 分组汇总 | 按指定维度分组后求和、平均、计数等聚合计算 |
| 数据筛选 | 按条件过滤数据行 |
| 字段设置 | 调整字段显示、排序、重命名，支持添加计算字段 |

**数据工厂的处理结果可以作为仪表盘的数据源**，形成"表单采集 → 数据工厂处理 → 仪表盘展示"的完整数据链路。这是帆软 BI 基因在产品内的直接体现。

### 3.5 企业互联

解决**跨组织协作**问题（如供应商对账、经销商协同、外包服务商对接）。

| 能力 | 详情 |
|------|------|
| 互联通讯录 | 构建外部组织通讯录，设置外部企业对接人 |
| 动态外部权限 | 外部人员只能访问被授权看到的数据 |
| 跨平台互联 | 16 种互联方式，兼容 PC 和移动端 |
| 消息触达 | 钉钉、企业微信、微信、飞书、短信、邮件多通道 |

### 3.6 知识库

结构化沉淀组织的制度规范、业务知识与员工经验。

| 能力 | 详情 |
|------|------|
| 文档创建 | 在线编辑 + Word 文档导入 |
| 嵌入式仪表盘 | 知识库中嵌入仪表盘，阅读手册时即可提交表单数据 |
| 权限管理 | 灵活的文档权限 + 团队分享 |
| 目录导航 | 结构化目录，支持多级分类 |
| 导出 | 支持 PDF 导出 |

### 3.7 模板生态

简道云提供 **200+ 行业模板**，覆盖主流业务场景：

- **通用**: OA 审批、任务管理、日报周报、资产管理
- **生产制造**: 生产报工、设备巡检、质量检验、仓库管理
- **人事行政**: 招聘管理、入转调离、考勤、培训
- **销售 CRM**: 线索跟进、客户管理、合同管理
- **项目管理**: 任务看板、工时统计、项目预算
- **疫情防控**: 健康打卡、来访登记、物资管理

模板生态是简道云快速获客和降低落地门槛的核心手段——用户可一键安装模板，30 分钟内完成基础应用搭建。

## 4. AI 能力

简道云从 2024 年开始逐步引入 AI 能力，目前主要围绕**表单填报辅助**和**流程自动化**两个方向。

### 4.1 AI 助理

面向企业成员的办公提效助手，提供两种核心能力：

| 能力 | 说明 |
|------|------|
| **AI 填充** | 自动填写表单字段内容，帮助节约成员填写时间，实现标准化数据录入。如：根据客户名称自动填充公司信息、根据产品名称自动填充规格参数 |
| **AI 对话填报** | 通过自然语言对话一站式完成表单填报。用户描述需求，AI 自动理解并填入对应字段 |

### 4.2 智能助手 Pro

规则驱动的自动化工具，在 AI 加持下升级为可调用 AI 能力的编排引擎。

| 能力 | 说明 |
|------|------|
| 触发方式 | 表单提交、修改、定时、Webhook 触发 |
| 执行动作 | 修改数据、发送通知、调用插件、调用 AI 模型 |
| 场景 | CRM 线索自动分配回收、人事入转调离自动流转、库存自动补货提醒 |

### 4.3 Agent 节点（智能助手 Pro 内）

简道云引入了 **Agent 节点**，具备自主理解目标、规划决策、执行行动的能力。Agent 可以依据用户指令自主调用插件工具直至完成目标。当前支持的工具体系主要是内置插件。

### 4.4 AI 实验室（公测中）

简道云在 AI 实验室中提供多项探索性 AI 功能供用户试用反馈，为正式功能上线提供迭代依据。

### 4.5 AI 能力评估

与明道云的对比：

| 维度 | 简道云 | 明道云 |
|------|--------|--------|
| AI 表单填写 | ✅ AI 填充 + AI 对话填报 | ❌ 无原生支持 |
| AI Agent | ✅ Agent 节点（插件调用） | ✅ AI Agent 工作流工具调用 + MCP |
| AI 文本生成 | ✅ 基础文本生成 | ✅ 支持 |
| RAG 知识库 | 🔲 未见公开文档 | ✅ 支持 RAG 知识库 |
| AI 集成深度 | 浅层（辅助填写为主） | 深层（工作流嵌入 AI 决策节点） |

**简道云的 AI 策略偏向"实用辅助"**——降低表单填写门槛，提升数据录入效率。明道云的 AI 策略更"激进"——在工作流中嵌入 AI 作为决策节点。AUDEBase 的 AI 集成策略（Phase 4 规划）可以借鉴两者的平衡。

## 5. 开放与集成

### 5.1 API 体系

简道云提供标准 RESTful API，覆盖以下资源：

| API 模块 | 覆盖接口 |
|----------|---------|
| 应用接口 | 应用创建、配置管理 |
| 表单接口 | 表单结构查询、字段配置 |
| 数据接口 | 数据 CRUD、批量操作、条件查询、附件上传下载 |
| 工作流接口 | 流程发起、流程查询、任务处理 |
| 通讯录接口 | 成员管理、部门查询、角色管理 |
| 文件接口 | 文件上传下载 |
| CRM 接口 | 客户、线索、商机管理 |

### 5.2 Webhook

简道云提供事件驱动的 Webhook 回调机制：

| 触发事件 | 说明 |
|----------|------|
| 数据提交 | 表单数据新增时推送到外部系统 |
| 数据修改 | 表单数据更新时推送变更 |
| 数据删除 | 数据删除时推送删除事件 |
| 数据恢复 | 数据从回收站恢复时推送 |
| 表单结构变化 | 表单字段增加/修改/删除时推送 |

### 5.3 插件市场

简道云通过代码封装实现了一系列插件，只需简单配置便可实现复杂对接场景。

| 指标 | 数值 |
|------|------|
| 插件分类 | 10 类 |
| 插件总数 | 100+ |
| 扩展方式 | 平台开发预置 + 用户反馈需求 + 自建插件 |

**插件开发模式**: 如果插件市场中没有满足需求的插件，用户可向简道云提交需求反馈，或自行开发插件。但与 NocoBase/AUDEBase 的"一切皆插件"微内核架构不同，简道云的插件系统是**附加扩展层**而非核心架构。

### 5.4 办公平台集成

简道云的**核心竞争优势之一**是与中国主流办公平台的深度集成：

| 平台 | 集成深度 |
|------|---------|
| 钉钉 | 原生集成：组织架构同步、单点登录、消息通知、工作台入口 |
| 企业微信 | 原生集成：通讯录同步、SSO、应用消息推送 |
| 飞书 | 原生集成：组织架构 + SSO + 消息通知 |
| 微信 | 微信服务号/小程序入口，支持微信支付 |

### 5.5 前端事件

简道云在表单填写时支持**前端事件**——实时调用外部接口处理数据。例如：

- 输入身份证号时自动调用公安接口验证并填充性别/出生日期
- 输入企业名称时自动查询工商信息填充
- 扫描条码时自动查询商品库返回品名/规格/价格

### 5.6 集成架构模式

简道云支持双向集成，但以**单向集成**为主（简道云作为中心枢纽，其他系统向简道云推送或从简道云拉取数据）。真正的双向实时同步需要开发者通过 API + Webhook 组合实现。

## 10. AUDEBase 可借鉴点

### 10.1 应当借鉴的设计与策略

#### ✅ 表单驱动作为"快速启动层"

简道云证明了：**80% 的企业数字化场景仅需要"表单 + 流程 + 报表"即可覆盖**。AUDEBase 虽然定位为模型驱动平台，但应该在 Schema Engine 之上提供表单驱动的**快速搭建模式**作为低门槛入口，类似 NocoBase 的 UI Editor。

**借鉴方案**: Schema Engine 的 ProTable/ProForm 映射器（D7）天然支持声明式表单生成。AUDEBase 可以提供预设模板和快速搭建向导，降低非开发人员的使用门槛。

#### ✅ 数据采集 → 处理 → 可视化全链路闭环

简道云的核心竞争力在于**完整的"表单采集 → 数据工厂处理 → 仪表盘展示"数据闭环**。这是帆软 BI 基因的直接体现。

**借鉴方案**: AUDEBase 应在 Phase 2 Schema Engine 阶段设计数据管道能力，Phase 4 引入轻量 BI（仪表盘 + 图表联动），形成"Schema 建模 → API 采集 → 数据聚合 → 可视化消费"的完整数据链路。不需要对标 FineBI 的深度分析能力，但应覆盖 80% 的日常报表需求。

#### ✅ 模板生态降低落地门槛

简道云 200+ 模板是一大获客利器——用户一键安装即可 30 分钟搭建应用。

**借鉴方案**: AUDEBase 的插件市场（Phase 2）应同时承载"功能插件"和"业务模板插件"两类资产。业务模板插件包含预定义的 Schema、流程、仪表盘配置，降低企业用户——特别是 OA/ERP 场景——的落地门槛。

#### ✅ 与国内办公生态深度集成

简道云与钉钉/企微/飞书的原生集成是许多企业选型的决定性因素。

**借鉴方案**: AUDEBase 在 Phase 1 管理 UI 中应考虑 OA 场景的办公平台集成接口预留。Phase 2+ 提供与钉钉/企微/飞书的 SSO、消息通知、通讯录同步插件。

#### ✅ 等保认证 + 私有化部署的安全合规路径

简道云通过等保三级 + 独享版私有化部署满足了中大型企业和政府客户的安全合规需求。这是国内 toB 产品不可回避的准入门槛。

**借鉴方案**: AUDEBase 从 Day 1 关注安全合规（D8.1 JWT 密钥管理、D12 Core 代理防止绕过权限），Phase 2+ 应规划等保认证路径和私有化部署方案。

#### ✅ 自负盈亏的健康商业模式

简道云 2017 年盈亏平衡、不依赖 VC 烧钱的路径，证明了 toB 产品可以走"健康增长"路线。

**借鉴方案**: AUDEBase 的开源 + 商业插件模式（参考 NocoBase）可以借鉴简道云的自负盈亏理念——通过开源获取社区信任和开发者生态，通过商业插件和托管服务实现可持续收入。

### 10.2 应当避免的局限

#### ❌ 表单驱动的架构天花板

简道云的表单驱动架构在复杂多表关联业务（如 ERP MRP 运算、MES 工序调度）面前力不从心。这是表单驱动模式的**根本性局限**——表单本质上是二维表结构的映射，无法原生表达实体间复杂关系。

**AUDEBase 应对**: 坚持模型驱动（Collection + Field 元数据），在模型之上提供表单驱动的快速搭建模式作为轻量入口，而非将表单作为底层数据模型。

#### ❌ 封闭架构不可扩展

简道云的插件系统是官方控制的附加层，而非 NocoBase 式的"一切皆插件"开放架构。第三方无法深度扩展核心功能。

**AUDEBase 应对**: 坚持微内核 + 插件热插拔（D1），所有核心功能以插件形式实现，确保第三方可以深度扩展。

#### ❌ 国际化缺失

简道云仅支持中文，没有多语言、多时区能力，限制了出海企业的使用。

**AUDEBase 应对**: 从 Day 1 实现国际化（D14/D15），双命名空间 + react-i18next，支持多语言。

#### ❌ 开发者生态薄弱

简道云的核心用户群体是业务人员，对开发者友好度低。开发者工具的缺乏限制了复杂集成和定制化需求。

**AUDEBase 应对**: 保持"开发者优先"的产品理念，提供完整的 API、CLI 工具、SDK、插件开发框架。

#### ❌ AI 策略偏保守

简道云的 AI 能力目前集中在"辅助填写"层面，未将 AI 作为核心差异化节点。明道云在 AI Agent + MCP 方面走得更远。

**AUDEBase 应对**: Phase 4 的 AI 集成应定位为"AI 协作层"，类似 NocoBase v2 的 AI 员工模式——将 AI 嵌入 Schema 建模、流程构建、数据查询等开发和管理环节。

### 10.3 差异化机会总结

| 简道云的局限 | AUDEBase 的机会 |
|-------------|----------------|
| 表单驱动 → 复杂业务支持弱 | 模型驱动 + Schema Engine → 支撑 ERP/MES/PLM |
| 封闭架构 → 不可深度扩展 | 微内核 + 一切皆插件 → 无边界扩展 |
| 仅中文 → 出海受限 | 国际化 Day 1 → 全球化市场 |
| 开发者不友好 → 定制困难 | 开发者优先 → API/CLI/SDK/插件框架 |
| AI 保守 | Phase 4 AI 协作层 → 开发全流程 AI 辅助 |
| 闭源 SaaS → 合规顾虑 | 开源 + 私有化 → 安全可控 |
