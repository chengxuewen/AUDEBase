## 1. 产品概述

### 1.1 一句话定位

**APaaS 零代码超级应用平台**，以业务对象建模为核心，私有部署成熟度国内领先，AI Agent + MCP Server 为企业应用提供智能决策与开放集成能力。

### 1.2 基本信息

| 维度 | 详情 |
|------|------|
| 产品名称 | 明道云 HAP (Hyper Application Platform)，前身为"明道"协作平台 |
| 所属公司 | 明道 (上海万企明道软件有限公司) |
| 创始团队 | 任向晖 (创始人/CEO) |
| 产品定位 | 超级应用平台 — 覆盖零代码构建、超自动化、数据集成、云原生部署、AI 智能 |
| 发布时间 | 2019 年正式发布零代码平台 (V10.0 起更名为 HAP) |
| 当前版本 | V12.2 (截至 2026 年 4 月) |
| Gartner 认可 | 低代码开发代表性产品 |
| 投资背景 | 曾获亿元级投资 |
| 融资轮次 | A 轮及以后 |
| 生态规模 | 500+ 合作伙伴，近百万私有部署用户，4000+ 付费企业 |
| 典型客户 | 中铁三局、佛山市铁路投资、迪卡侬、四川航空、东方证券、华夏银行、沃尔玛、可口可乐、艾默生、大兴机场 |

### 1.3 发展历程

| 时间 | 里程碑 |
|------|--------|
| 2019 年 | 零代码平台正式发布 |
| 2020 年 | 私有部署版本成熟，开放前端代码开源 (mingdao-openweb) |
| 2022 年 | 集成中心上线，支持实时数据同步 |
| 2023 年 11 月 | V10.0 — 产品更名为 HAP 超级应用平台 |
| 2024 年 3 月 | SaaS 定价首次大幅调整 (五年来首次) |
| 2025 年 10 月 | V12.0 — AI Agent + 对话机器人 + MCP Server 上线 |

| 2026 年 4 月 | V12.2 — 内置向量知识库 + AI Action 上线 |

### 1.4 产品理念

明道云选择 **No-Code APaaS (模型驱动)** 技术路径，对标 Zoho Creator、Airtable、ServiceNow。核心理念：

- **"No Code, No Bug"** — 一次对，次次对；一处对，处处对
- **公民开发 (Citizen Development)** — 让业务人员而非开发者构建应用
- **可组合性 (Composability)** — 通过灵活组合能力颗粒完成高复杂度企业应用
- **开放生态** — 产品完全退出解决方案和实施服务市场，全部由合作伙伴交付

### 1.5 愿景

> "超级应用平台" — 零代码构建企业应用只是起点，围绕它发展超自动化引擎、应用数据集成、云原生私有部署、AI 智能、插件架构，以及开发与实施生态。

## 2. 技术架构

### 2.1 架构概览

明道云系统整体基于 **微服务架构**，前端通过 HTTP 连接 API Gateway，API 与微服务、微服务与微服务之间基于 **gRPC** 实现 RPC 调用。

```
┌──────────────────────────────────────────────────────────┐
│                    前端层 (Frontend)                       │
│  React (Web/H5) — 可视化设计器 — 响应式多端适配            │
│  私有部署前端代码完整开源 (mingdao-openweb)                 │
├──────────────────────────────────────────────────────────┤
│                    API Gateway                            │
│         HTTP 接入 — 自动生成 RESTful API — 鉴权            │
├──────────────────────────────────────────────────────────┤
│              微服务层 (Microservices)                      │
│  核心服务: 工作表引擎 / 工作流引擎 / 权限引擎 / 通知服务     │
│  集成服务: 集成中心 (Flink ETL) / Webhook / API 连接        │
│  AI 服务: Mingo / AI Agent / 对话机器人 / 向量知识库        │
│  RPC: gRPC (微服务间通信)                                  │
├──────────────────────────────────────────────────────────┤
│              数据存储层 (Data Storage)                     │
│  MySQL      — 用户、组织等系统基础数据                       │
│  MongoDB    — 业务数据 (应用、工作表、工作流等)              │
│  Redis      — 缓存 (支持 Redis Cluster 分片)               │
│  Kafka + ZooKeeper — 消息队列 (工作表汇总/收集/工作流事件)   │
│  Elasticsearch — 全局搜索                                  │
│  MinIO      — 文件存储 (默认，支持 S3 兼容对象存储)          │
│  Flink      — 数据集成实时同步与 ETL                        │
└──────────────────────────────────────────────────────────┘
```

### 2.2 技术栈明细

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端框架 | React | Web 端 + H5 移动端 |
| 后端语言 | Java, C#, Node.js | 多语言混合微服务 |
| RPC 协议 | gRPC | API ↔ 微服务 + 微服务间通信 |
| API 协议 | RESTful (自动生成) | 所有工作表自动生成 CRUD API |
| 关系型数据库 | MySQL | 用户、组织、系统配置 |
| 文档数据库 | MongoDB | 业务数据主存储 (应用/工作表/工作流) |
| 缓存 | Redis | 支持 Cluster 分片、哨兵模式 |
| 消息队列 | Kafka + ZooKeeper | 工作表汇总、工作流事件 |
| 搜索引擎 | Elasticsearch | 协作套件全局搜索 |
| 文件存储 | MinIO (默认) | 支持 S3 标准 (阿里云 OSS/腾讯云 COS/华为云 OBS 等) |
| 流处理 | Apache Flink | 数据集成实时同步 + ETL |
| 容器化 | Docker / Docker Compose | 单机部署 |
| 编排 | Kubernetes | 集群高可用部署 |
| 分库分表 | ShardingSphere | 海量数据扩展 |
| 信创替代 | 巨杉 SequoiaDB, TapDB | 可替代 MongoDB |

### 2.3 元数据驱动架构

核心设计：**配置即应用，模型即代码**

- 所有用户配置 (表单、权限、流程) 以 JSON 格式存储为元数据
- 运行时动态解析并渲染，生成可执行逻辑
- 用户自定义的数据模型实时生成数据库 Schema，无需手动编码
- 无源代码输出 — 纯粹的解释运行模式

### 2.4 部署架构

#### 单机模式 (Docker Compose)

- `ops-allinone` 单一镜像，通过 `ROLE` 环境变量区分服务角色
- 最低硬件要求: 4C8G CPU/内存，100G 磁盘
- 适合: 中小企业、POC 验证、开发测试

#### 集群模式 (Kubernetes)

- 微服务容器化部署，支持弹性伸缩
- MongoDB 副本集 (自动故障转移)
- Redis Cluster 分片 (自动分片 + 智能路由)
- 适合: 中大型企业、高并发场景

#### 多数据中心多活 (V6.4+)

- 生产中心 + 同城灾备 + 异地灾备 三层架构
- 专线实时数据同步
- 故障自动切换，业务零中断
- 适合: 金融、政务、制造等对连续性要求极高的行业

### 2.5 安全机制

| 维度 | 实现 |
|------|------|
| 传输加密 | TLS 1.3 |
| 存储加密 | AES-256 |
| 多租户隔离 | Tenant ID 逻辑隔离 + Schema 分离 |
| 权限模型 | RBAC + 字段级三态权限 |
| 二次验证 | 短信/邮箱 OTP |
| 渗透测试 | 定期执行 |
| 前端开源 | 私有部署版 Web 前端代码完整开源 (mingdao-openweb) |
| 数据库审计 | MongoDB 副本集 + 完整操作日志 |

## 3. 核心功能

HAP 通过 **8 个功能模块** 组合构建企业应用：

### 3.1 工作表 (Worksheet)

核心数据建模工具，对标数据库表：

- **字段类型**: 60+ 种 (文本、数字、日期、附件、关联记录、子表、定位、签名等)
- **关联记录**: 一对一、一对多跨表关联，支持级联删除/更新
- **数据容量**: 标准版 10 万行/表，专业版 100 万行，旗舰版不限
- **数据验证**: 字段级校验规则 + 公式计算
- **导入导出**: Excel 批量导入/导出，支持模板

### 3.2 视图 (View)

数据展示与操作界面：

- **视图类型**: 表格视图、看板视图、日历视图、画廊视图、甘特图
- **过滤与排序**: 多条件组合过滤，自定义排序
- **字段配置**: 按视图配置显示/隐藏字段
- **公开视图**: 生成外部分享链接，支持密码保护

### 3.3 角色与权限 (Roles & Permissions)

- **权限模型**: RBAC (基于角色的访问控制)
- **字段级权限**: 三态控制 — 隐藏 / 可见只读 / 可见可编辑
- **数据隔离**: 按角色配置记录可见范围 (例如：销售员仅见自己的客户)
- **多租户**: Tenant ID 逻辑隔离 + Schema 分离
- **外部用户**: 外部门户独立权限体系

### 3.4 工作流 (Workflow)

超自动化引擎核心，支持：

- **触发方式**: 工作表记录新增/修改/删除、定时触发、Webhook 触发、按钮触发
- **流程控制**: 条件分支、并行分支、子流程、循环
- **审批**: 会签、或签、逐级审批、转审、加签
- **自动化节点**:
  - 新增/更新/删除/查询记录
  - 发送通知 (站内/邮件/企微/钉钉/飞书)
  - 调用封装业务流程 (PBC — Packaged Business Capabilities)
  - 发送 API 请求 (RESTful, GET/POST/PUT/DELETE/PATCH)
  - 代码块 (Python/Node.js)
  - JSON 解析
  - AI Agent 节点 (V12.0+)
- **版本管理**: 工作流支持版本回滚
- **执行配额**: 标准版含一定免费额度，可增购 ¥10/1000 次·月

### 3.5 统计图表 (Statistics)

- **图表类型**: 柱状图、折线图、饼图、雷达图、漏斗图、数值图、透视表等
- **数据源**: 单表或多表关联
- **仪表盘**: 组合多个图表为自定义数据看板
- **下钻**: 支持点击图表下钻到明细记录
- **局限**: 深度分析能力弱于帆软/简道云，需外接 BI

### 3.6 自定义页面 (Custom Pages)

- **可视化搭建**: 拖拽式页面设计器
- **组件**: 富文本、图片、按钮、嵌入视图/图表/工作流触发
- **布局**: 自由布局 + 响应式适配
- **用途**: 仪表盘首页、业务展示页、操作面板
- **扩展**: 私有部署版前端代码开源，支持 React 自定义开发

### 3.7 外部门户 (External Portal)

- 面向外部用户 (客户/供应商/合作伙伴) 的独立访问入口
- 独立的权限体系和用户管理
- 支持自定义品牌和域名
- 外部用户包: 100 人 ¥500/年，1000 人 ¥1,000/年

### 3.8 对话机器人 (Chatbot) — V12.0+

- 基于 AI Agent 的对话式业务助手
- 支持自然语言查询数据、执行业务操作
- 可结合知识库提供智能问答
- 支持嵌入到企微/钉钉/飞书等协作平台

## 4. AI 能力全景

明道云 V12.2 确立了 **"AI×产品双赋能"** 战略，AI 不再是外挂模块，而是 HAP 的底层驱动力。

### 4.1 Mingo 全能助手

- 按 M 键随时呼出
- **搭建辅助**: 自然语言描述自动生成工作表、视图、统计图表、工作流、权限配置
- **数据操作**: 对话式查询、录入、更新业务数据
- **内容生成**: 智能生成审批意见、周报、合同草稿
- **完全免费**: Token 费用由明道云承担
- **演进方向**: 未来将整合对话机器人能力，成为统一入口

### 4.2 AI Agent (工作流节点)

明道云最核心的企业级 AI 能力 — 将 AI 做成可拖拽的工作流节点：

- **自然语言理解**: 读取表单内容、工单描述、合同文本，自动提取关键信息
- **智能判断与分支**: 自动分类工单、判断报销合规性、识别客户意向 → 触发不同流程
- **自动生成内容**: 生成周报、合同草稿、审批意见、客户跟进邮件
- **工具调用**: Agent 可在工作流中自主调用平台工具 (CRUD/通知/API/封装业务流程)
- **权限控制**: 字段级、工作表级权限 — AI 只能访问授权数据

### 4.3 对话机器人 (AI Chatbot)

以 Agent 为核心的业务应用：

- **多轮自然语言交互**: 像聊天一样查询客户数据、项目进度、库存、财务报表
- **自动执行业务操作**: 发起审批、创建客户、更新订单、分配任务
- **知识库问答**: 结合内置向量知识库精准回答
- **权限可控**: 按用户权限隔离数据
- **会话分享**: 支持生成链接分享，可设置有效期与密码

### 4.4 向量知识库 (V12.2 新增)

- **文档智能检索**: 上传 PDF、Word、Excel，语义理解检索
- **跨应用调用**: 工作流、AI Agent、对话机器人可跨应用检索知识库
- **动态引用上下文**: AIGC 节点可引用整条工作表记录，AI 理解更精准

### 4.5 AI Action

- 记录详情页一键触发 AI 场景对话
- 基于当前记录上下文，直接输出总结、分析、建议

### 4.6 AI 与传统 AI 工具对比

| 对比维度 | 传统 AI 工具 (ChatGPT 等) | 明道云 AI |
|----------|--------------------------|-----------|
| 数据来源 | 公共数据/手动输入 | 企业私有业务数据 (工作表 + 知识库) |
| 业务集成 | 独立工具，需手动复制数据 | 原生嵌入业务流，自动读写数据 |
| 权限控制 | 弱，难隔离敏感数据 | 企业级字段/表级权限，安全可控 |
| 自动化能力 | 无，需人工执行结果 | AI Agent 驱动端到端自动化流程 |
| 使用门槛 | 需 Prompt 技巧 | 零代码，自然语言直接操作 |

### 4.7 AI 路线图

- AI 超级搜索 — 对所有应用数据向量化，语义搜索
- Mingo 全能助手 — 整合搭建 + 使用 + 分析
- AI 总结字段 — 编写提示词让 AI 归纳记录数据
- AI 设计建议 — 基于知识库给出配置建议
- AI 直接生成应用 — 分步确认、全程可调整

## 5. MCP Server 生态

明道云是目前**国内唯一将 MCP Server 作为正式产品能力发布的低代码平台**。

### 5.1 mdymcp — 统一 MCP Server

- **PyPI 包**: `mdymcp` (当前版本 v0.4.0)
- **安装方式**: `uv tool install mdymcp` → 交互式配置向导
- **工具数量**: 98 个工具
  - **v1 协作 API (50 个)**: 动态/日程/私信/收件箱/群组/用户/组织/个人账户
  - **HAP 网关 (48 个)**: 应用/工作表/记录/角色成员/工作流审批/图表/选项集/知识库/地区组织

### 5.2 支持的 AI IDE/客户端

| IDE/客户端 | 配置方式 | 支持范围 |

| Windsurf | `~/.codeium/windsurf/mcp_config.json` | 用户级 |

### 5.3 架构设计

```

              ┌───────────▼──┐ ┌──▼───────────┐
```

### 5.4 认证机制

| 类型 | 机制 |
|------|------|
| v1 API | 本地 OAuth → access_token + refresh_token (自动续期) |
| HAP 网关 | Personal Access Token (PAT, `pat_` 开头) — 长期有效，随时可吊销 |
| 安全 | v1 token 全程本地 (chmod 600)，HAP PAT 无远端交换 |

### 5.5 典型 AI 使用场景

- **Cursor 中对话式操作业务数据**: "帮我查一下张三的订单记录" → AI 直接调用 HAP 工具
- **Dify 工作流中调用 HAP**: 构建智能客服 → 用户问题 → 检索知识库 → 更新工单
- **Claude Code 中自动化运维**: 批量创建应用、同步配置、导出数据

### 5.6 第三方生态扩展

社区合作伙伴基于 MCP 开发了 **"超级智能体"视图插件**：
- MCP 插件可视化配置管理
- 智能对话 + 工具调用 + 工作表数据联动
- 全链路对话历史追溯
- 已应用于智能知识库 (RAG)、客服情感分析、供应链预测、HR 智能化

## 6. 私有部署体系

明道云的**核心竞争力之一**：国内最早公开提供容器化私有部署的零代码平台。

### 6.1 部署模式

| 模式 | 适用场景 | 技术栈 |
|------|----------|--------|
| **单机版 (Docker Compose)** | 中小企业、POC | ops-allinone 单一镜像 |
| **集群版 (Kubernetes)** | 中大型企业、高并发 | 微服务容器化 + MongoDB 副本集 + Redis Cluster |
| **多数据中心多活 (V6.4+)** | 金融、政务、制造 | 生产+同城灾备+异地灾备 |

### 6.2 最低硬件要求

- CPU: 4 核
- 内存: 8 GB
- 磁盘: 100 GB 可用空间

## 11. 优劣势分析

### 11.1 核心优势

1. **业务对象建模能力强** — APaaS 架构天然适合复杂多表关联场景，远超表单驱动平台
2. **AI 能力国内领先** — AI Agent 在工作流中可调用工具、对话机器人、向量知识库，形成完整 AI 矩阵
3. **MCP Server 独占** — 国内唯一将 MCP 作为正式产品能力，连接 Cursor/Dify/Claude Code 等 AI 客户端
4. **私有部署成熟度最高** — Docker Compose/K8s/多数据中心多活，文档完善，前端代码开源
5. **开放 API 体系** — 所有工作表自动生成 RESTful API + Webhook + 代码块扩展 + 数据集成 (HDP)
6. **生态伙伴模式** — 500+ 伙伴交付，产品团队聚焦产品本身
7. **前端开源** — 私有部署 Web 端代码完整开源 (React)，支持二次开发
8. **集成中心** — Flink 驱动的实时数据同步，支持多种数据库
9. **工作流强大** — 封装业务流程 (PBC) 可复用、条件分支/子流程/循环/AI Agent 节点

### 11.2 核心劣势

1. **报表能力弱** — 基础图表，深度分析必须外接 BI 系统 (简道云内建帆软 BI 完胜)
2. **价格较高** — 50 人标准版 ¥20,900，是简道云 (¥8,360) 的 2.5 倍
3. **学习曲线陡峭** — 业务对象建模需要理解实体关系，非技术人员上手难度高
4. **市场占有率** — SaaS 市占率不及简道云 (IDC 第一)，以私有部署差异化竞争
5. **模板生态不如简道云** — 简道云 200+ 行业模板可直接套用
6. **单表容量限制** — 标准版仅 10 万行/表，对大数据量场景需升级版本
7. **移动端体验** — H5 适配而非原生 App，体验有差距
8. **信创生态有限** — 虽然支持巨杉/TapDB，但整体信创认证和适配不如用友/金蝶

### 11.3 机会

- **AI + 低代码融合**: 明道云在 AI 布局上已领先国内竞品 1-2 个版本
- **MCP 生态**: 随着 MCP 协议成为 AI 工具交互标准，先发优势明显
- **国产替代**: 信创政策推动下，私有部署能力是核心壁垒
- **数据集成 (HDP)**: 独立产品化后，可拓展到非 HAP 客户

### 11.4 威胁

- **简道云 AI 追赶**: 帆软资金雄厚，可能快速补齐 AI 短板
- **大厂入局**: 钉钉宜搭、飞书低代码依托生态优势快速渗透
- **开源低代码**: NocoBase、Appsmith 等开源方案侵蚀私有部署市场
- **价格战**: 算数云等新玩家以 1/10 价格切入，压低行业利润

## 12. AUDEBase 可借鉴点

### 12.1 APaaS 业务对象建模

**借鉴**: 实体 → 字段 → 关联关系 → 视图 → 工作流 的完整链路

**对 AUDEBase 的启示**:
- 明道云的"业务对象"概念与 AUDEBase 的 Schema Engine (D3) 高度对应
- 参考其"工作表 = 业务对象"的抽象层级 — 用户不需要理解数据库
- 关联记录字段 (一对一/一对多) 是复杂建模的核心竞争力

### 12.2 AI Agent + 工作流深度融合

**借鉴**: AI Agent 作为工作流节点，可调用平台工具 (CRUD/通知/API)

**对 AUDEBase 的启示**:
- AUDEBase 工作流引擎 (Phase 4) 应预留 AI Agent 节点接口
- AI 调用平台能力的关键: 权限隔离 — Agent 只能访问授权数据
- 参考其"工具调用"模式: Agent 在工作流中自主选择调用哪个平台能力

### 12.3 MCP Server — 应用即 AI 工具

**借鉴**: 将应用接口暴露为标准化 MCP 工具，供外部 AI 客户端调用

**对 AUDEBase 的启示**:
- 这比传统 REST API 更进一步 — AI 可以直接"使用"你的应用
- AUDEBase 应考虑从 Day 1 设计 MCP 兼容接口
- 参考其 98 个工具的分类: 协作 API (通用) + HAP 网关 (业务)
- mdymcp 的交互式安装向导设计值得借鉴 — 降低配置门槛

### 12.4 私有部署成熟度

**借鉴**: Docker Compose (单机) → K8s (集群) → 多数据中心多活 三阶段部署体系

**对 AUDEBase 的启示**:
- AUDEBase 应从 Phase 1 就支持 Docker Compose 单机部署
- 运维平台独立部署 — 不与主应用耦合
- 前端代码开源策略 — 既保护商业核心，又满足定制需求
- 最低硬件要求文档化 (4C8G/100G)

### 12.5 字段级三态权限

**借鉴**: 隐藏/可见只读/可见可编辑 三态控制

**对 AUDEBase 的启示**:
- AUDEBase 的字段级权限 (D11) 应直接采用此三态模型
- 比简单的"可见/不可见"二元模型更贴近企业实际需求
- 需要在前端 (Schema → UI 自动隐藏/禁用) 和后端 (API 过滤) 同时实现

### 12.6 封装业务流程 (PBC)

**借鉴**: 将工作流编排的数据处理流程封装为可复用的"业务流程组件"

**对 AUDEBase 的启示**:
- 这是 Gartner PBC (Packaged Business Capabilities) 思想在零代码平台的实践
- AUDEBase 的工作流引擎应考虑"子流程封装 + 参数化调用"
- 类似于函数编程中的"高阶函数" — 将流程作为一等公民

### 12.7 元数据驱动 + 多数据库混合

**借鉴**: MySQL (系统) + MongoDB (业务) + Redis (缓存) + Elasticsearch (搜索) + Flink (ETL)

**对 AUDEBase 的启示**:
- 不追求"一种数据库解决所有问题"，而是按职责选择最优存储
- AUDEBase 当前决策用 Drizzle ORM + PostgreSQL (D9)，但未来可考虑 MongoDB 做业务数据的灵活存储
- 多模态数据库的专业化优势：各司其职，整体最优
- 用户不需要与原生技术栈打交道 — 零代码屏蔽复杂度

### 12.8 商业模式 — 纯产品 + 伙伴交付

**借鉴**: 明道云完全退出解决方案和实施服务市场，专注产品本身

**对 AUDEBase 的启示**:
- 开源/开放核心 + 生态伙伴交付 是可持续的商业模式
- 产品团队聚焦产品，避免陷入定制开发的泥潭
- 前端代码开源降低伙伴二次开发门槛

### 12.9 数据集成独立产品化 (HDP)

**借鉴**: 将数据集成能力从 HAP 剥离为独立产品，覆盖更广泛的系统生态

**对 AUDEBase 的启示**:
- AUDEBase 的数据集成能力可以考虑独立模块化 — 不绑定平台
- 工作流风格的同步管道配置 — 降低使用门槛
- 支持手动/定时/实时多种同步模式
